PConsole

Tutorial

This tutorial will cover the basics of using PConsole, a language created by me for manipulating variables, performing calculations, and more. It is (hopefully) designed to be simple and easy to use. It can either be written using the console or in a .cfg file. The console is useful for testing out code and running small snippets, while .cfg files are better for writing larger blocks of code and saving them for later use. You can run a .cfg file by either specifying the command in the default.json file or by running PConsole.exe in a terminal with the argument -file "path1/path2/path3/file.cfg". To start, try running the help command: PConsole help() This will display a list of the methods, variables, and namespaces in the root namespace. There is also the "Debug.Dump" command which will display everything in every namespace. You can run multiple lines at once by separating them with a semicolon.

An important thing to note is that you can create comments (or temporarily disable code) by either typing out // which will comment out the rest of the line or putting /* and */ around everything you want to comment out. PConsole // This whole line is a comment /* This is a comment */ But this isn't

Variables

Let's then try creating a variable. To create a variable, you can use the methods in the "Var" namespace. For example, to create a variable named "x" with the value 5, you can use the following command: PConsole Var.Declare("x", 5) You can change the value of an existing variable like this: PConsole x = 10 You are only allowed to declare variables in the root namespace, but variables can exist in other namespaces when set in the default environment. The default environment or "Base Space" is set in the BaseSpace.json file. The default file can also be changed using the default.json file.

Now that we have a variable, we can access its value by putting a dollar sign in front of its name. We can print things to the console using the echo command: PConsole echo($x) Variables are hard typed. This means you cannot set a variable that is already a string to a have a value that is a number. If you want to change the type of a variable you can use a := operator instead of a normal equals sign, or use the Var.Override command. The type of a variable is just set when it is defined, to whatever value was given to it. There are four types currently available. Strings, booleans, integers, and floats. You can also make lists of variables, which can be any type. Skip this part if you have ever used a programming language before. Strings are called strings because they are essentially a string of characters. A character is just anything you can type out on a computer (the alphabet, numbers, punctuation, and even spaces and new lines. You can create a string by typing out a quotation mark " and then any text, closed with another quotation mark. You might notice that you can't put a quotaion mark in a string but there is a solution that we will get to later. Booleans are just true or false. They can be used for things like comparing numbers. The value of $a == $b is either true or false. We will come back to comparing values later. Numbers can either be an integer (or Int32 as they are called internally) or floats (or Singles). Integers can only be whole numbers whereas float can either be whole numbers or decimal numbers. You can convert betweem them as much as you want but converting a decimal number to an integer will cut off the decimal part. To make a float, you can just add a decimal point to a number. For example, 3.14 is a float, while 3 is an integer. You can also add an "f" to the end of a number to make it a float, like this: 3f. This is useful if you want to make sure a number is treated as a float even if it doesn't have a decimal point. PConsole type(3); type(3f); You can make a list just by putting square brackets around a list of variables, separated by commas. Each element can have its own type but are hard-typed like normal variables. Try creating a variable that is a list of a string, a boolean, an integer, and a float: PConsole Var.Declare("x", ["hi", true, 10, 3.14]); // create the variable echo($x[0]); // print out the value of the first element type($x[1]); // print out the type of the second element x[2] = 20; // change the value of the third element x[3] := "overrided" // override the value of the fourth element

Methods

Methods are just a way of simplifying code. Instead of writing the same code over and over, you can make it a method so you only need to write out the name of the method instead of the whole block. You can create your own methods with the "alias" command: PConsole alias("sayHello", { echo("Hello!") }); sayHello() If you write your code in a .cfg file instead, you can put new-lines between lines of code: PConsole alias("sayHello", { echo("Hello!") }); sayHello()

Methods can also take arguments. Arguments are just variables that are only used in the method. They are defined in the parentheses after the name of the method. To access them in our alias we can use %0 for the first argument, %1 for the next, and so on. For example, we can make a method that takes a name as an argument and says hello to that name: PConsole alias("sayHello", { echo("Hello " + %0) }); sayHello("Dappermonke") Methods can also return values, which will always be the output of their last line. PConsole alias("add", { %0 + %1 }); echo(add(1, 2));
